## 1. Executive Summary

This document presents an **updated research plan** for the Adelic Constraints on Quantum Field Theory project, incorporating findings from the comprehensive literature review (0.4.md) and the critical analysis of the original 10-module plan (0.3.md). The updated plan:

1. **Retains** 6 of the original 10 modules (restructured and refined)
2. **Adds** 4 new modules (pre-registration, null models, generalized q-adic framework, experimental prediction synthesis)
3. **Removes or defers** 2 original modules (QED beta function derivation, adelic Standard Model Lagrangian)
4. **Restructures** the execution order into 4 sequential phases with explicit validation gates
5. **Incorporates** the generalized $q$-adic framework from the existing corpus (beyond prime-restricted $p$-adic)
6. **Adds** falsifiable prediction targets and statistical validation throughout

### 1.1 Core Research Question (Refined)

> Can dimensionless physical constants — specifically the low-energy fine-structure constant $\alpha \approx 1/137.036$ — be constrained or derived from the requirement that physical partition functions, amplitudes, or correlation functions are consistently defined across all completions of $\mathbb{Q}$ (the adelic framework), and if so, with what precision?

> **⚠️ AMENDED by 0.8.md:** The success criteria below target $\alpha(0) \approx 1/137$ — a running coupling at one scale, not a fundamental invariant. See 0.8.md for revised criteria targeting the **adelic beta function** $\beta(\alpha)$ and the **cross-ratio as a function of scale**.

### 1.2 Success Criteria (Pre-Registered)

Before any code is written, we define what would constitute evidence for and against the adelic hypothesis:

| Outcome | Criterion |
|:--------|:----------|
| **Strong positive** | A closed-form expression for $\alpha$ derived from adelic constraints that matches the experimental value to within current experimental uncertainty ($\pm 1.5 \times 10^{-10}$) |
| **Moderate positive** | An adelic constraint that selects a rational frequency $\omega$ whose numerical value agrees with $\alpha^{-1}$ or a related physical ratio to within $\sim 1\%$, with the agreement improving as computational precision increases |
| **Weak positive** | Identification of number-theoretic patterns (e.g., in zeta zero spacings, adelic partition function structure) that correlate with physical mass ratios or coupling constants at statistically significant levels ($p < 0.01$ after multiple comparison correction) |
| **Null result** | No statistically significant patterns emerge; adelic products show no structure that correlates with known physical constants |
| **Negative result** | Adelic products diverge or are undefined without arbitrary regularization, indicating that the naive adelic factorization hypothesis is mathematically inconsistent |

#### Module 2: p-adic and q-adic Analysis — Partition Functions

**Status:** RETAINED with MODIFICATIONS from original Module 2.

**Objective:** Implement Haar integration on $\mathbb{Q}_p$ and generalized $\mathbb{Q}_q$, compute classical partition functions $Z_p(\beta)$ and $Z_q(\beta)$, and establish convergence properties.

**Key modifications:**

| Modification | Rationale |
|:-------------|:----------|
| **Defer Vladimirov derivative** to a later module (or remove) | The literature review shows the Vladimirov operator is mathematically subtle; the partition function can be computed without it using the classical approximation |
| **Add q-adic partition function** | Generalization to arbitrary $q$ per corpus findings |
| **Add convergence analysis** | Before combining into adelic products, verify that individual $Z_p(\beta)$ series converge and characterize the convergence rate as a function of $p$ |
| **Add real partition function with frequency parameter** | $Z_\infty(\beta, \omega) = \sqrt{\pi/(\beta\omega)}$ with explicit $\omega$ dependence for the adelic product |

**Core computation:**
- $p$-adic classical partition function: $Z_p(\beta, \omega_p) = 1 + \sum_{n=1}^\infty p^{-n}(1-p^{-1}) e^{-\beta \omega_p p^{2n}}$

- Convergence analysis: compute $Z_p(\beta, 1)$ for $p = 2, 3, 5, 7, 11, \ldots, 97$ and determine how rapidly $Z_p \to 1$ as $p \to \infty$

**Validation gate:** Before proceeding, verify that:
1. $Z_p(\beta) \to 1$ as $p \to \infty$ (ensuring adelic product convergence)
2. $Z_p(\beta) \to 1$ as $\beta \to \infty$ (unit ball dominates at low temperature)
3. The series formula matches direct numerical integration of $\int_{\mathbb{Q}_p} e^{-\beta|x|_p^2} d\mu(x)$ for at least one test case

#### Module 3: Adelic Partition Function — Structure & Constraints

**Status:** SIGNIFICANTLY MODIFIED from original Module 3.

**Objective:** Construct the adelic partition function $\Xi(\beta, \omega) = Z_\infty(\beta,\omega) \prod_p Z_p(\beta,\omega_p)$ and systematically study its structure. **Do not assume** $\Xi = 1$; instead, characterize the function's behavior and identify natural constraints.

**Critical change from original plan:** The original plan searches for $\omega$ such that $\Xi(1,\omega) = 1$. However, the literature review and existing analysis reveal that:

1. The infinite product over primes may **diverge** without regularization
2. There is no a priori reason why the product should equal 1 — the product formula $\prod_v |q|_v = 1$ is for **norms**, not partition functions
3. The constraint $\Xi = 1$ is a physical **hypothesis**, not a mathematical necessity

**Revised approach — Three-stage investigation:**

**Stage 3a: Product Convergence Study**
- Compute truncated products $\Xi_N(\beta, \omega) = Z_\infty \prod_{p \leq P_N} Z_p$ for increasing $P_N$
- Determine whether $\Xi_N$ converges as $N \to \infty$
- If divergent, characterize the divergence (logarithmic? power-law?)
- Test regularization schemes: ratio $\Xi(\beta_1)/\Xi(\beta_2)$, zeta regularization, etc.

**Stage 3b: Adelic Product Landscape**
- Compute $\Xi(\beta, \omega)$ over a grid of $(\beta, \omega)$ values
- Identify natural features: minima, maxima, saddle points, level sets
- Plot the "adelic landscape" as a 2D heatmap

**Stage 3c: Natural Constraint Identification**
- Instead of imposing $\Xi = 1$, ask: what mathematical principle would naturally constrain this product?
- Candidates: modular invariance, scale invariance, self-duality under $\beta \to 1/\beta$, requirement that $\Xi$ be an automorphic form
- Test each candidate constraint and see what values of $\omega$ they select
- Compare selected $\omega$ values with $\alpha^{-1} \approx 137.036$

**Validation gate:** Before proceeding to Phase II, the convergence behavior of the adelic product must be fully characterized. If the product diverges for all regularization schemes, the entire adelic partition function approach must be reconsidered — this is the most critical validation gate in the entire project.

**Deliverables:**
- `src/adelic_product.py` — $\Xi(\beta, \omega)$ computation with truncation control
- `module_03_report.md` — Convergence study, landscape analysis, constraint candidates
- `images/03_convergence.png` — Truncated product vs $P_N$
- `images/03_landscape.png` — 2D heatmap of $\Xi(\beta, \omega)$
- `images/03_constraint_omega.png` — Selected $\omega$ under various constraints

**Estimated effort:** 3–4 sessions (most critical module)

#### Module 5: Hierarchical RG & Adelic Fixed Points

**Status:** RETAINED with SCOPE REDUCTION from original Module 5.

**Objective:** Implement the hierarchical RG recursion on a $p$-adic tree (Bethe lattice), compute fixed points, and explore whether adelic combinations of fixed points yield physically meaningful critical exponents.

**Scope reduction:** The original plan aimed to derive critical exponents matching known exact values (e.g., 2D Ising $\nu = 1$). This is extremely ambitious. The revised scope:

1. Implement a **single, well-documented** recursion map $f_p$ for $\phi^4$ theory on the Bethe lattice
2. Compute fixed points $\lambda_*^{(p)}$ for $p = 2, 3, 5, 7$
3. Compute the real RG fixed point $\lambda_*^{(\infty)}$ (e.g., from $\epsilon$-expansion or exact 2D Ising)
4. Explore simple adelic combinations: product $\prod_p \lambda_*^{(p)}$, sum $\sum_p \lambda_*^{(p)}$, etc.
5. Compare resulting numbers with known physical quantities (critical exponents, coupling ratios)

**Key disclaimer:** This module explores *whether* adelic combinations of RG fixed points correlate with physical quantities. It does not claim to *derive* critical exponents from first principles. The distinction is important for scientific integrity.

#### Module 9: Null Models & Statistical Validation

**Status:** NEW module (not in original plan).

**Objective:** Construct null models to test whether any apparent "discoveries" from Modules 3–8 could arise by chance. This is essential for scientific integrity.

**Rationale:** The original plan had no null models. Without them, any apparent pattern (e.g., a zeta zero spacing matching a mass ratio) could be a statistical coincidence. The literature review (0.4.md, §5.3) identified this as a critical methodological gap.

**Null models to construct:**

| Null Model | Tests |
|:-----------|:------|
| **Random rational frequencies** | Generate random $\omega = a/b$ and compute $\Xi(\beta,\omega)$. Does the actual search over structured rationals produce results distinguishable from random? |
| **Shuffled physical constants** | Shuffle the digits of $\alpha$, mass ratios, etc. and repeat the symbolic regression search. Does the real $\alpha$ produce better fits than shuffled versions? |
| **Random primes** | Replace actual primes with random integers in the adelic product. Does the prime structure matter? |
| **GUE surrogate** | Generate surrogate zeta "zeros" from the GUE distribution and repeat the mass ratio fitting. Do actual zeta zeros perform better? |
| **Cross-ratio from random points** | Generate random 4-tuples of points in $\mathbb{P}^1$ and compare their cross-ratios with the "special points" cross-ratios from Module 7 |

**Validation gate:** Before claiming any positive result, the corresponding null model must be rejected at a pre-specified significance level (e.g., $p < 0.01$ after correction).

**Deliverables:**
- `src/null_models.py` — Implementation of all null models
- `module_09_report.md` — Null model results, corrected significance levels

## 4. Execution Strategy

### 4.1 Per-Module Workflow

For each module, the LLM agent follows this standard workflow:

1. **Pre-flight check:** Read the module specification; verify all prerequisites are satisfied
2. **Code development:** Write the Python script(s) in `src/`
3. **Execution & debugging:** Run the code via the UI execution environment; fix errors iteratively
4. **Output capture:** Save all printed output, generated data, and plots
