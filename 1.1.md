## 1. Project Overview & Motivation

### 1.1 The Central Question

> Can the renormalization group flow of dimensionless physical couplings — specifically the fine-structure constant's beta function $\beta_{\text{QED}}(\alpha)$ — be derived or constrained by the requirement that physical amplitudes are consistently defined across all completions of the rational numbers $\mathbb{Q}$?

### 1.2 The Running-Coupling Correction

**The original framing was "can we derive $\alpha \approx 1/137$ from adelic constraints?" This is the wrong question.** The fine-structure "constant" runs with energy: at the Thomson limit ($\mu = m_e$), $\alpha \approx 1/137.036$; at the Z-mass, $\alpha(M_Z) \approx 1/127.9$; at the Planck scale, it would be different still. The low-energy value is a **section of a function evaluated at one point** — not a fundamental invariant. Seeking a closed-form expression for $\alpha(0)$ is analogous to seeking a closed form for the temperature at a specific weather station without modeling the atmosphere.

**The correct target is the beta function:**

$$\beta(\alpha) \equiv \mu \frac{d\alpha}{d\mu}$$

This encodes HOW $\alpha$ changes with scale. The known one-loop QED beta function is:

$$\beta_{\text{QED}}(\alpha) = \frac{2\alpha^2}{3\pi} \sum_f Q_f^2 + \mathcal{O}(\alpha^3)$$

The deep number-theoretic mystery is not "why $1/137$?" but "why **this** beta function?" — why the $\alpha^2$ leading term, why these specific coefficients, why this functional form?

### 1.3 Ostrowski's Theorem and the Adelic Framework

**Ostrowski's Theorem:** The only non-trivial completions of $\mathbb{Q}$ (up to equivalence) are:
- The real numbers $\mathbb{R}$ (Archimedean, $|\cdot|_\infty$)
- The $p$-adic numbers $\mathbb{Q}_p$ for each prime $p$ (non-Archimedean, $|\cdot|_p$)

Together these form the **adele ring** $\mathbb{A}_\mathbb{Q}$. The **adelic product formula** states that for any non-zero rational $q$:

$$\lvert q \rvert_\infty \prod_p \lvert q \rvert_p = 1$$

This is the only universal constraint relating Archimedean and non-Archimedean measures. If physical laws can be expressed over $\mathbb{Q}$, this constraint must be respected.

### 1.4 The Generalized Ratio Framework

Prior work (Quni-Gudzinas, 2025–2026; see 0.4.md) generalizes beyond integer primes $p$ to arbitrary scaling ratios $q$ (including transcendental values $\pi$, $e$, $\phi$, $\alpha^{-1}$). The **Bruhat-Tits tree** $T_q$ provides the geometric substrate for each $q$-adic completion. The **Monna projection** $\Phi_q: \mathbb{Z}_q \to [0,1]$ maps the ultrametric tree boundary onto the Archimedean line — a lossy, many-to-one projection that explains apparent randomness, quantum indeterminacy, and prime distribution as **projection artifacts**.

### 1.5 The Scale-Dependent Cross-Ratio

Every real number is a cross-ratio of four collinear points in $\mathbb{P}^1(\mathbb{R})$ (established in 0.1.md). In the adelic setting, those four points can be **idele-valued** — energy scales. Their cross-ratio becomes a **function** on the idele class group, not a single number.

For the electromagnetic coupling, the natural four points are mass scales:

$$\chi(\mu) = (m_e, m_\mu; m_\tau, \mu)$$

where $\mu$ is the observation scale. The "running" of $\alpha$ is the variation of the Archimedean component of this adelic cross-ratio as $\mu$ moves along a path in the idele class group. The beta function emerges as:

$$\beta(\alpha) = f'(\chi) \cdot \mu \frac{d\chi}{d\mu}$$

where $\alpha = f(\chi)$ for some function $f$ determined by the adelic constraint.

### 1.6 All Running Couplings as Cross-Ratios

The framework extends naturally to the full Standard Model:

| Coupling | Interpretation | Scale Dependence |
|:---------|:---------------|:-----------------|
| $\alpha$ (electromagnetic) | Cross-ratio of lepton mass scales + observation scale | Logarithmic running |
| $\alpha_s$ (strong) | Cross-ratio of QCD scales | Asymptotic freedom |

The adelic product formula may constrain this entire system of cross-ratios simultaneously.

### 3.2 Key Formulas

**Product formula** (for $q \in \mathbb{Q}^\times$):
$$\lvert q \rvert_\infty \prod_p \lvert q \rvert_p = 1$$

**$p$-adic classical partition function** (harmonic oscillator, $V(x) = \lvert x \rvert_p^2$):
$$Z_p(\beta, \omega_p) = 1 + \sum_{n=1}^\infty p^{-n}(1-p^{-1}) e^{-\beta \omega_p p^{2n}}$$

**Real classical partition function** ($V(x) = \omega x^2$):
$$Z_\infty(\beta, \omega) = \sqrt{\frac{\pi}{\beta\omega}}$$

**Adelic partition function**:
$$\Xi(\beta, \omega) = Z_\infty(\beta, \omega) \prod_p Z_p(\beta, \omega_p)$$

where $\omega_p = \lvert \omega \rvert_p$ for rational $\omega$.

**Adelic product formula (Freund-Witten, 1987)**:
$$\mathcal{A}_\infty(s,t) \prod_p \mathcal{A}_p(s,t) = 1$$

for the Veneziano amplitude at specific kinematic points $(s,t)$.

**Cross-ratio**:
$$(a,b;c,d) = \frac{(a-c)(b-d)}{(a-d)(b-c)}$$

**QED beta function (one-loop)**:
$$\beta_{\text{QED}}(\alpha) = \frac{2\alpha^2}{3\pi} \sum_f Q_f^2$$

#### Module M8: Adelic Beta Function Reconstruction

**Objective:** Using prime-by-prime discrete RG maps from M4 (Veneziano amplitudes) and the cross-ratio flow framework from M7, reconstruct the full adelic beta function — the discrete map at each prime and the Archimedean interpolation — and predict $\alpha(M_Z)$ from a high-scale adelic boundary condition.

**Why this is better than the original "search for $\alpha$":** The original plan searched for a number ($\alpha \approx 1/137$). A number can match by coincidence. This plan searches for a **functional relationship** — the beta function $\beta(\alpha)$ — which is a much higher-dimensional target, far less susceptible to coincidental matches.

**15-minute cap strategy:** The search space is too large for 15 minutes. Use **checkpointing**: the script saves its state to `data/08_checkpoint.json` every $\sim$10 minutes. Resume from checkpoint. Alternatively, partition the pre-registered search space into independent sub-executions.

**Core investigations:**

| Investigation | Description |
|:--------------|:------------|
| I8.1 | **Prime-by-prime beta assembly:** For each prime $p$, extract a discrete RG transformation $\alpha \to \alpha_p'$ from the $p$-adic Veneziano amplitude (M4 data). Assemble the discrete adelic RG map. |
| I8.2 | **Archimedean interpolation:** Interpolate between discrete prime RG steps to obtain a continuous $\beta(\alpha)$. Fit to a polynomial in $\alpha$. Compare coefficients with known one-loop, two-loop, three-loop QED beta function. |
| I8.3 | **Boundary condition from adelic product formula:** Impose $\mathcal{A}_\infty \prod_p \mathcal{A}_p = 1$ on the four-point amplitude. This determines the high-scale boundary condition $\alpha(\Lambda_{\text{Pl}})$. Integrate downward to predict $\alpha(M_Z)$. |
| I8.4 | **Parallel beta functions:** Repeat the reconstruction for $\alpha_s$ (strong coupling) and $\sin^2\theta_W$ (Weinberg angle). Investigate whether the adelic product formula imposes relations between these beta functions. |
| I8.5 | **Prediction vs experiment:** Compare the predicted $\alpha(M_Z)$ with the measured value $\alpha^{-1}(M_Z) = 127.952 \pm 0.009$. Report the prediction with explicit uncertainty from truncation, interpolation error, and boundary condition ambiguity. |

**Validation gate G8:** The reconstructed beta function must match the **functional form** of the known Standard Model beta functions — specifically, the correct powers of $\alpha$ in the leading terms. The predicted $\alpha(M_Z)$ from the adelic boundary condition must agree with experiment within the estimated uncertainty of the reconstruction. Any candidate that survives must also survive cross-validation and null model testing (M9).

**Deliverables:**
- `src/prime_beta.py` — prime-by-prime discrete RG maps
- `src/adelic_beta.py` — adelic beta function assembly and interpolation
- `src/rg_integration.py` — RG integration from high scale to Z-mass

- `data/08_checkpoint.json` — incremental search state

#### Module M9: Null Models & Statistical Validation

**Objective:** Construct null models to test whether apparent "discoveries" from M3–M8 could arise by chance. This is essential for scientific integrity and prevents confirmation bias.

**Rationale:** Without null models, any apparent pattern (a zeta zero spacing matching a mass ratio, a cross-ratio approximating $\alpha$) could be a statistical coincidence. The null models quantify the false-positive rate.

**Null models:**

| Null Model | Tests |
|:-----------|:------|
| **Random rational frequencies** (M3) | Generate random $\omega = a/b$ and compute $\Xi(\beta,\omega)$. Compare the distribution of $\Xi$ values for structured rationals vs random. |
| **Random prime RG maps** (M8) | Replace each $f_p$ with random discrete transformations. Does the reconstructed beta function still match QED? |
| **Shuffled coefficient vectors** (M8) | Shuffle the prime-by-prime coefficients. Does the interpolation still produce a smooth beta function? |
| **Random boundary condition** (M8) | Use random high-scale $\alpha(\Lambda_{\text{Pl}})$ values. How sensitive is $\alpha(M_Z)$ to the boundary condition? |
| **GUE surrogate** (M6) | Generate surrogate zeta "zeros" from the GUE distribution and repeat the mass ratio fitting. |
| **Random cross-ratios** (M7) | Generate random 4-tuples of points in $\mathbb{P}^1$ and compare their cross-ratios with those from special points. |

**Validation gate G9:** Before claiming any positive result from M3–M8, the corresponding null model must be rejected at a pre-specified significance level ($p < 0.01$ after Bonferroni correction for multiple comparisons across all tested hypotheses).

**Deliverables:**
- `src/null_models.py` — all null model implementations
- `module_09_report.md` — null model results, corrected significance levels
- `data/09_null_results.json` — distributions, $p$-values
- `images/09_null_comparison.png` — real vs null distributions

**Estimated effort:** 2–3 sessions
**Runtime:** 5 independent null-model runs $\times$ $\sim$5–10 min $\approx$ 25–50 min total

## 7. Execution Strategy

### 7.1 Per-Module Workflow

For each module, the LLM agent:

1. **Pre-flight:** Read the module specification; verify prerequisites satisfied
2. **Code:** Write Python script(s) in `src/`
3. **Execute:** Run via the UI execution environment. For modules with runtime > 15 min, use the batching/checkpointing protocol (§7.2)
4. **Debug:** Fix errors iteratively
5. **Capture:** Save all printed output, generated data, and plots
6. **Report:** Compose `module_XX_report.md` following the standard template (0.7.md §5)
7. **Validate:** Verify gate criteria are met
8. **Commit:** Git commit with Conventional Commits message
9. **Review:** At phase end, review accumulated results before proceeding

### 7.2 15-Minute Hard Cap Protocol

**HARD CONSTRAINT:** No single code execution may exceed 15 minutes (900,000 ms).

| Module | Strategy |
|:-------|:---------|
| M1–M5, M7 | Single execution — all run in < 5 min |
| M6 (zeta zeros) | **Batch:** 10 independent executions of 10K zeros each ($\sim$10 min each) + 1 merge |
| M8 (beta reconstruction) | **Checkpoint:** Save state to `data/08_checkpoint.json` every $\sim$10 min. Resume with `--resume`. Total: 4–12 incremental executions |
| M9 (null models) | **Independent runs:** 5 null models as separate executions ($\sim$5–10 min each) |
| M10 (synthesis) | Single execution |

**Checkpoint file format:**

```json
{
  "search_space": {"constants": [...], "operations": [...], "max_depth": 8},
  "completed_combinations": 1247,
  "best_candidates": [{"expression": "...", "fitness": 0.9987, "eval_count": 3}],
  "total_combinations": 10000,
  "last_checkpoint_time": "2026-05-09T12:00:00Z",
  "random_seed": 42
}
```

**Background execution with polling:**

```python
exec(command="python src/zeta_analysis.py --batch 0 --batch-size 10000",
     description="M6: Compute zeta zeros batch 0",
     background=True, timeoutMs=900000)
process(action='poll', sessionId='bg_XXXX')
```

### 7.3 Go/No-Go Decision Points

**After Phase I (M3):**
- **GO** if: $\Xi_N$ converges (with or without regularization) and shows non-trivial structure
- **NO-GO** if: $\Xi_N$ diverges unconditionally for all regularization schemes
- **Fallback:** Proceed with M4–M8 (Veneziano, zeta, cross-ratio) which do not depend on $\Xi$ converging

**After Phase III (M8):**
- **GO** if: Reconstructed $\beta(\alpha)$ matches QED functional form; $\alpha(M_Z)$ prediction within estimated uncertainty
- **DOCUMENT NULL RESULT** if: No reconstruction possible — the null result is scientifically valuable
